## 2 证明（Proofs）

问：什么是证明？

答：一种确定真理的方法。

举例：

- 实验/观察（物理学）
- 抽样（统计学）
- 法律（法官/陪审团）
- 公众舆论
- 商业（权威）
- 学校（教授说的！）
- 宗教（神的话语）
- 内心信念
- 「为什么不呢？」

在数学中：

**定义 1**：一个数学证明（mathematical proof）是对一个命题（proposition）的验证，通过从一组基本公理（axioms）出发，经过一系列逻辑推导（logical deductions）来完成。

接下来的两节课中，我们将深入探讨**谓词逻辑**（predicate logic，也称为一阶逻辑），分解这个定义，并理解这三个加粗部分的含义。

:::tip
你可能会想：「我是来学编程的，为什么要学数学证明？」事实上，证明的思想渗透在计算机科学的每个角落：

- **程序正确性**：你写的排序算法真的对所有输入都能正确工作吗？这需要证明。
- **算法分析**：为什么快速排序是 $O(n \log n)$？这需要数学推导。
- **密码学**：RSA 加密为什么安全？因为它依赖于「大数分解在计算上是困难的」这一数学事实。
- **AI/机器学习**：一个模型是否总能收敛到最优解？这需要理论保证。

本课程将教你如何像计算机科学家一样严谨地思考——这是写出正确、高效代码的基础。
:::

## 3 命题（Propositions）

**定义 2**：一个命题（proposition）是一个要么为真（True）要么为假（False）的陈述。

**定义 3**：一个谓词（predicate）是其真假依赖于变量取值的命题。

示例：

- **命题 1**（真）：$2 + 3 = 5$
- **命题 2**（假）：$2 + 3 = 6$
- **命题 3**：$\forall n \in \mathbb{N}. \; n^2 + n + 41$ 是素数

一些非命题的例子：

- 「你好。」
- 「你是谁？」
- 「这个陈述是假的。」（陈述引用自身意味着什么？并非总是有意义。本课程不处理这类问题。）

:::tip
你可以把**谓词**想象成编程中的函数，它接收参数，返回 `true` 或 `false`：

- `isPrime(n)` 就是一个谓词——输入 $n$，输出该数是否为素数。
- `greaterThan(x, y)` 也是谓词——输入两个数，返回 $x$ 是否大于 $y$。
- SQL 的 `WHERE age > 18` 本质上就是在用谓词筛选数据。

谓词是数学逻辑的核心构件，也是你后续学习数据库查询、类型系统、程序验证等领域的基础。现在花时间理解它，后面的内容会轻松很多。
:::

一些记号：

- $\forall$（「对于所有」）：全称量词（universal quantifier），后面跟谓词「$n^2 + n + 41$ 是素数」
- $\in$（「属于」）：集合中的元素
- $\mathbb{N}$（「自然数集」）：非负整数集合 $\{0, 1, 2, \ldots\}$

要判断命题 3 是真是假，我们需要知道对于每一个 $n$，谓词是否为真。我们来验证一下：

| $n$ | $n^2 + n + 41$ | 素数？ |
| :---- | ---------------- | :----: |
| 0     | 41               |   是   |
| 1     | 43               |   是   |
| 2     | 47               |   是   |
| 3     | 53               |   是   |
| 4     | 61               |   是   |
| 5     | 71               |   是   |
| ...   | ...              |  ...  |
| 20    | 461              |   是   |
| ...   | ...              |  ...  |
| 39    | 1601             |   是   |

前 40 个例子都是素数，所以很可能是真的，对吧？在某些领域，这已经足够令人信服：工程学、物理学、CS（模拟常常替代证明）。但当 $n = 41$ 时：

$$
n^2 + n + 41 = 41^2 + 41 + 41 = 43 \times 41
$$

在数学中，我们称之为「举例证明（Proof by Example）」，这**不是**有效的证明！

一些数论中的命题甚至更加狡猾。

**命题 4**（假）：$a^4 + b^4 + c^4 = d^4$ 无正整数解。

- 由 Euler 于 1769 年提出
- 最终被 Noam Elkies 推翻
- 解为：$a = 95800, b = 217519, c = 414560, d = 422481$

**命题 5**（真）：$\exists a, b, c, d \in \mathbb{Z}^+. \; a^4 + b^4 + c^4 = d^4$

**命题 6**（假）：$313(x^3 + y^3) = z^3$ 无正整数解。

- 最小的反例超过 1000 位数字
- 为什么关心这个？
  - 关于 $313(x^3 + y^3) = z^3$ 有什么意义？
  - 关于 1000 位数字有什么意义？
- 事实上，这是椭圆曲线（elliptic curve）的一个例子。椭圆曲线对于理解如何分解大数至关重要。分解大数的能力将使你能进入地球上大多数计算机系统……

**命题 7**（哥德巴赫猜想，Goldbach's Conjecture）：每个大于 2 的偶数都是两个素数之和。

- 真还是假？未知！
- 迄今为止检查过的每个偶数都成立……
- 普遍认为是正确的
- 似乎非常困难，曾在 90 年代中期被列为《波士顿环球报》头版未解之谜之一

## 4 组合命题（Combining Propositions）

通过组合已有命题来构造新命题是非常有用的。例如：

- $A$ **且** $B$，记为 $A \land B$，意思是「$A$ 和 $B$ 都为真」
- $A$ **或** $B$，记为 $A \lor B$，意思是「$A$ 或 $B$ 中至少有一个为真」（注意：这是**包含性或**（inclusive or），即 $A$ 和 $B$ 同时为真也是可以的）
- **非** $A$，记为 $\bar{A}$ 或 $\neg A$，意思是「$A$ 为假」

我们可以用真值表（truth tables）来表示这些：

| $A$ | $\neg A$ |
| ----- | ---------- |
| T     | F          |
| F     | T          |

| $A$ | $B$ | $A \land B$ | $A \lor B$ |
| ----- | ----- | ------------- | ------------ |
| T     | T     | T             | T            |
| T     | F     | F             | T            |
| F     | T     | F             | T            |
| F     | F     | F             | F            |

**警示故事**：数学术语是精确的，但英语词汇往往根据上下文改变含义，而二者共享大量术语。更糟的是，我们用英语来交流数学！当以公式书写时，$\lor$ 等符号始终对应上述真值表。但在日常语言中：

- 婚礼上的服务员说晚餐选择是「chicken or pasta」，这里的 "or" 传达的是一组允许的回应。这种情况下，你可以说「chicken」或「pasta」，但不能说「both」或「neither」。这不是我们定义的「或」，这是**异或（exclusive or），记为 $\text{xor}$
- 服务员接着说你可以要「coffee or tea」，这次你可以说「just coffee」、「just tea」或「neither」，但不能说「both」（因为你只有一个杯子，他们不会在高级派对上把两者混在一起！）。这又不是 $\lor$，这是「非且」（not and），有时称为 $\text{nand}$
- 给你的咖啡，服务员说可以加「cream or sugar」。这次，四种选项都可以！英语单词 "or" 的三种用法，没有一次真正对应逻辑「或」

## 5 蕴含（Implication）

另一个重要的布尔运算符：$A$ 蕴含 $B$**（$A$ implies $B$），记为 $A \to B$ 或 $A \Rightarrow B$。应读作「$A$ 蕴含 $B$」，即「如果 $A$，那么 $B$」。

| $A$ | $B$ | $A \Rightarrow B$ |
| ----- | ----- | ------------------- |
| T     | T     | T                   |
| T     | F     | F                   |
| F     | T     | T                   |
| F     | F     | T                   |

「F $\Rightarrow$ T 为真」这一行可能不太直观，让我们具体化一下：「星期三我们穿粉色」，即「如果是星期三，就穿粉色」，即「星期三蕴含粉色」。现在 F $\Rightarrow$ T 为真就说得通了：星期四穿粉色并不违反规则。

**注意**：$A \Rightarrow B$ 等价于它的逆否命题（contrapositive），即 $(\neg B) \Rightarrow (\neg A)$，但不等价于它的逆命题（converse） $B \Rightarrow A$。

一些让人抓狂的例子：

- 你可能熟悉 `<3`，表情符号出现之前的「爱心」，即「我爱你」。我也见过 `<4` 作为加强版，「我真的爱你」。但后者更弱！$x < 3$ 蕴含 $x < 4$，但反之不成立，所以 $x < 3$ 是更强（即信息量更大）的陈述。也许应该是 `<2`？
- 笛卡尔：「我思故我在。」这句话断定了 $T$，也断定了 $T \Rightarrow A$。网络迷因：「我不思，故我不在。」这句话断定了 $\neg T$ 以及 $\neg T \Rightarrow \neg A$。这两个蕴含并不等价！这是反命题（inverse），等价于逆命题，但不等价于原命题。如果我的鞋存在但不思考，笛卡尔对此没问题，但那个迷因就不对了。应该是「我不在，故我不思」？

**注意**：$A \Rightarrow B$ 并不意味着因果关系（「$A$ 导致 $B$」）或与时间有任何关联（「$A$ 发生了，随后导致 $B$ 发生」）。$A \Rightarrow B$ 仅由上述真值表定义。

顺便说一句，如果 F $\Rightarrow$ T 这一行被换成我们之前可能期望的值，那就成了「**当且仅当**」（if and only if），缩写为 $A \text{ iff } B$，它断言 $A$ 和 $B$ 要么同为真，要么同为假。它等价于 $(A \Rightarrow B) \land (B \Rightarrow A)$，这也是它名称的由来。

## 6 小插曲：集合（Sets）

粗略地说，一个集合（set）是一组对象的汇集。可以是有限的（如 $A = \{6, 1, 2, 0\}$）、无限的（如 $\mathbb{N} = \{0, 1, 2, \ldots\}$，$\mathbb{Z} = \{\ldots, -2, -1, 0, 1, 2, \ldots\}$）、空的（空集记为 $\varnothing$ 或 $\{\}$，含义相同）、可以包含其他集合作为元素（$B = \{2, \{3, 4\}, \varnothing\}$）等等。

集合中**没有重复元素**（有重复元素的叫多重集 multiset），元素的**顺序无关紧要**。因此 $\{1, 2, 2, 3\} = \{2, 3, 1\} = \{3, 1, 2\}$。

集合 $A$ 有 4 个元素，$B$ 有 3 个元素。$6 \in A$ 表示 6 是 $A$ 的元素。$\{1, 2\} \in A$？（否）$\{3, 4\} \in B$？（是）

集合之间可以有子集（subset）关系：$S \subseteq T$ 表示 $S$ 的所有元素也是 $T$ 的元素。$\{1, 2\} \subseteq A$？（是）$\{0, 1, 2, 6\} \subseteq A$？（是）

使用谓词来指定子集通常很有用，例如：

$$
\{n \in \mathbb{N} \mid \text{isPrime}(n)\} = \{2, 3, 5, 7, 11, \ldots\}
$$

这称为集合构造式记号（set-builder notation）。有时也会用冒号代替竖线。

将集合组合成其他集合也很有用：

- 交集（intersection）：$A \cap B = \{2\}$
- 并集（union）：$A \cup B = \{0, 1, 2, 6, \{3, 4\}, \varnothing\}$
- 差集（difference）：$A - B = A \setminus B = \{0, 1, 6\}$（在 $A$ 中但不在 $B$ 中的元素）

相比之下，当顺序重要时，我们有有序元组（ordered tuples），用圆括号表示：$X = (6, 1, 2, 0)$ 与 $Y = (2, 1, 6, 0)$ 不同。重复元素也是允许的，如 $(6, 1, 2, 0, 0)$。并集、交集等对有序元组意义不大。

## 7 公理（Axioms）

证明的第二个组成部分是它所基于的公理集合。

**定义 4**：一个公理（axiom）是一个被假定为真的命题。

- 你可能听过：「数学中不要做假设」
- 你必须做假设！
- 关键在于提前声明它们。

:::tip
可以把数学想象成一个游戏。**公理**就是游戏的规则。不同的规则产生不同的游戏：

- **欧几里得几何**假设「过线外一点恰有一条平行线」——这就是中学课本里的几何。
- **非欧几何**假设「没有平行线」或「有无穷多条平行线」——初看很奇怪，但爱因斯坦的广义相对论用的就是非欧几何！

关键不在于规则本身"对不对"，而在于规则是否自洽（一致），以及能推导出什么有趣的结论。计算机科学中的各种形式系统（如类型系统、程序逻辑）也是基于同样的思想：先选好公理，再推导定理。
:::

示例：

- **公理 1**：如果 $a = b$ 且 $b = c$，则 $a = c$
- **公理 2**：给定一条直线 $l$ 和一个不在其上的点 $p \notin l$，恰好有一条通过 $p$ 且平行于 $l$ 的直线
- **公理 3：给定一条直线 $l$ 和一个不在其上的点 $p \notin l$，没有**通过 $p$ 且平行于 $l$ 的直线
- **公理 4**：给定一条直线 $l$ 和一个不在其上的点 $p \notin l$，有**无穷多条通过 $p$ 且平行于 $l$ 的直线

后三条公理相互矛盾！哪一条是对的？全都是同等有效的。**不同的公理产生不同的证明和定理。任何同意你公理的人都必须接受从这些公理导出的定理。在这三个世界中，「直线」只是不同的对象……

并不存在「正确」的公理集合，但我们至少希望：

**定义 5**：一个公理集合是一致的（consistent），如果没有命题可以同时被证明和被否证。

**定义 6**：一个公理集合是完备的（complete），如果每个命题都可以被证明或被否证。

一致性和完备性显然很理想！逻辑学家 Russell 和 Whitehead 穷其一生试图找到一组一致且完备的基本算术公理。但是：

定理 8（哥德尔不完备定理，Gödel's Incompleteness Theorem）：不存在同时完备且一致的适合算术的公理集合。

- 由 Kurt Gödel 在 1930 年代证明
- 震动了整个领域
- 对 Russell 和 Whitehead 而言是糟糕的一天……
- 推论：如果你想要一致性（这是必要的），那么就存在无法被证明的真命题！
- 也许：哥德巴赫猜想为真，但没有证明！
- 如果我们万一布置了这类题目作为作业，请告诉我们……
