## 1.3 什么是 C++？

### 1.3.1 一段合法的 C++ 程序

```cpp
#include <iostream>
#include <string>

int main() {
    auto str = std::make_unique<std::string>("Hello World!");
    std::cout << *str << std::endl;
    return 0;
}

// 输出 "Hello World!"
```

此程序包含了 C++ 的多个现代特性：模板（Templates）、智能指针（Smart Pointers）、流（Streams）、操作符重载（Operator Overloading）。

### 1.3.2 这也是一段合法的 C++ 程序（C 风格）

C++ 向后兼容 C 语言。

```cpp
#include "stdio.h"
#include "stdlib.h"

int main(int argc, char *argv[]) {
    printf("%s", "Hello, world!\n");
    // ^一个 C 函数！
    return EXIT_SUCCESS;
}
```

### 1.3.3 这甚至也是一段合法的 C++ 程序（汇编风格）

```cpp
#include "stdio.h"
#include "stdlib.h"

int main(int argc, char *argv[]) {
     asm(".LC0:\n\t"
                ".string \"Hello, world!\"\n\t"
           "main:\n\t"
                "push rbp\n\t"
                "mov rbp, rsp\n\t"
                "sub rsp, 16\n\t"
                "mov DWORD PTR [rbp-4], edi\n\t"
                "mov QWORD PTR [rbp-16], rsi\n\t"
                "mov edi, OFFSET FLAT:.LC0\n\t"
                "call puts\n\t");
     return EXIT_SUCCESS;
}
```

## 1.4 C++ 的历史

### 1.4.1 汇编语言时代

```asm
section .text
global _start
_start:
    mov edx,len
    mov ecx,msg
    mov ebx, 1
    mov eax, 4
    int 0x80
    mov eax, 1
    int 0x80
section .data
    msg db 'Hello, world!' ,0xa
    len equ $ - msg
```

汇编语言的优缺点：

| 优点 | 缺点 |
|------|------|
| 指令极其简单 | 代码量巨大（即使简单任务） |
| 速度极快（写得好时） | 难以理解 |
| 对程序拥有完全控制 | 极度不可移植 |

### 1.4.2 C 语言的发明

- 1972 年由 Dennis Ritchie 创造了 C 语言
- C 语言使编写**快速**、**简单**、**跨平台**的代码变得容易
- 编译器将源码翻译成汇编

然而 C 语言也有一些弱点：
- 没有对象和类
- 难以编写泛型代码
- 编写大型程序较为繁琐

### 1.4.3 C++ 的诞生

1983 年，丹麦计算机科学家 Bjarne Stroustrup 开始创造 C++。他想要一种语言兼具：
- **快速**
- **简单易用**
- **跨平台**
- **拥有高级语言特性**

## 1.5 C++ 设计哲学

核心原则（由 Bjarne Stroustrup 总结）：

1. **可读性 (Readable)**：直接在代码中表达想法和意图
2. **安全性 (Safety)**：尽可能在编译时强制执行安全规则
3. **效率 (Efficiency)**：不浪费时间和空间
4. **抽象 (Abstraction)**：将混乱的结构隔离封装
5. **程序员选择权 (Programmer Choice)**：允许程序员拥有完全的控制、责任和选择

> "代码应该优雅且高效；我讨厌在两者之间做选择。" — Bjarne Stroustrup

## 📚 补充知识点

### C++ 与 C 的关系

C++ 最初被称为"带类的 C"（C with Classes）。Bjarne Stroustrup 在设计 C++ 时的一个重要目标就是保持与 C 的兼容性。这意味着几乎所有的 C 程序都可以用 C++ 编译器编译通过。这种向后兼容性是一把双刃剑：一方面它使得庞大的 C 代码库可以平滑过渡到 C++，另一方面它也继承了 C 的一些不安全的特性。

### C++ 标准版本的关键特性

| 版本 | 年份 | 关键特性 |
|------|------|----------|
| C++98 | 1998 | 第一个国际标准，包含 STL |
| C++11 | 2011 | auto、lambda、智能指针、移动语义、`nullptr` |
| C++14 | 2014 | 泛型 lambda、`std::make_unique` |
| C++17 | 2017 | 结构化绑定、`std::optional`、`std::variant` |
| C++20 | 2020 | 概念（Concepts）、范围（Ranges）、协程（Coroutines） |
| C++23 | 2023 | 改进的范围、`std::expected`、更多 constexpr |

### C++ 编译器生态

三大主流 C++ 编译器：

1. **GCC (g++)**：GNU 编译器套件，开源且跨平台
2. **Clang (clang++)**：LLVM 项目的一部分，提供优秀的错误信息
3. **MSVC (cl.exe)**：微软的 Visual C++ 编译器，集成在 Visual Studio 中

### 编译 vs 解释

C++ 是编译型语言，这意味着源代码在运行前会完全转换为机器码。这与解释型语言（如 Python）形成对比，后者在运行时逐行解释执行。编译型语言通常具有更好的运行时性能，但开发周期中多了一步编译过程。

### C++ 在各行业的实际应用

通过学习 C++，你将能够接触到：
- 游戏引擎开发（Unreal Engine 使用 C++）
- 量化金融和高频交易系统
- 嵌入式系统（从微控制器到航天器）
- 操作系统内核开发
- 浏览器引擎（Chrome 的 Blink、Firefox 的 Gecko）
- 高性能计算（HPC）和科学计算
